# BlogAlone开发路线

这份路线用于约束MVP开发顺序。每个阶段完成后再进入下一个阶段，避免业务代码早于基础设施落地。

## 开发阶段

- [x] 阶段1搭建工程骨架和依赖闭环
  - 建立CMake/vcpkg工程、lib+exe目标、测试目标和基础目录。
  - 锁定Drogon、SQLite、libsodium、cmark-gfm、spdlog、GoogleTest依赖。
  - 编译并运行`deps_smoke`，确认核心依赖可链接、可调用。
  - 编译`blogalone`，启动最小Drogon服务，验证`GET /api/healthz`。

- [x] 阶段2落地数据库迁移系统
  - 实现启动期迁移插件，读取`schema_migrations`并按版本执行SQL。
  - 每个迁移使用`BEGIN IMMEDIATE`，记录版本、校验值和执行时间。
  - 启动时配置SQLite连接的`foreign_keys`、`WAL`、`synchronous`和`busy_timeout`。
  - 为迁移失败、重复执行、校验值变化补齐单元测试。

- [x] 阶段3建立后端公共基础设施
  - 定义统一错误类型、错误码和JSON响应格式。
  - 接入request_id生成、结构化日志和基础请求耗时记录。
  - 增加配置读取、UTC时间工具、随机token生成和SHA-256工具。
  - 建立HTML入口页、静态资源和上传文件访问的后端路由基础。
  - 建立控制器、服务、仓储之间的边界和最小接口约定。

- [x] 阶段4实现认证、会话和CSRF
  - 完成注册、登录、退出、`GET /api/me`和`PATCH /api/me`。
  - 使用libsodium Argon2id保存密码哈希。
  - 会话token和CSRF令牌只在Cookie和响应体暴露明文，数据库保存哈希。
  - 实现SessionFilter和CsrfFilter，覆盖Cookie属性、过期、撤销、封禁和写接口拒绝场景。

- [x] 阶段5实现板块、帖子和两层回复
  - 完成公开接口`GET /api/forums`、帖子列表、帖子详情。
  - 完成`/`和`/threads/:id`页面入口，页面数据通过JSON接口加载。
  - 完成登录用户发帖、回帖、楼中楼回复、编辑和删除自己的内容。
  - 回帖事务使用`db->newTransaction()`。
  - 列表接口统一分页结构，服务端限制`page`和`page_size`。

- [x] 阶段6实现Markdown安全渲染和图片上传
  - 接入cmark-gfm，渲染前遍历AST，删除原始HTML并限制链接协议。
  - 只允许白名单HTML标签，外链自动补`rel`属性。
  - 上传接口校验图片类型、文件头、尺寸、大小和每日额度。
  - 使用SHA-256去重，维护`uploads`和`upload_refs`，保存成功后绑定正文引用。
  - 上传文件访问由后端处理，拒绝路径穿越并按白名单MIME返回图片。

- [ ] 阶段7实现管理员能力
  - 完成板块创建、编辑、删除。
  - 完成置顶、加精、删除、恢复、封禁、管理员授权和撤权。
  - 高风险操作要求10分钟内二次密码确认。
  - 管理员操作写入`audit_log`，撤权时撤销目标用户有效会话。

- [ ] 阶段8补齐安全和运行维护任务
  - 实现注册、登录、上传、发帖的进程内滑动窗口限流。
  - 实现孤儿上传引用清理任务，删除超期未绑定文件。
  - 增加真实IP解析，只有可信代理的`X-Forwarded-For`会被采信。
  - 检查日志脱敏，禁止记录密码、Cookie、CSRF令牌和完整邮箱。

- [ ] 阶段9扩展测试覆盖
  - 服务层单元测试覆盖注册、登录、封禁、发帖、回帖、权限和Markdown清理。
  - 集成测试启动Drogon随机端口，覆盖页面路由、静态资源、HTTPAPI、Cookie、CSRF和数据库写入。
  - 针对并发回帖、迁移失败、上传异常和管理员授权加入回归测试。
  - 确保Windows开发环境和Linux部署环境都能完成依赖冒烟构建。

- [ ] 阶段10交付部署和备份流程
  - 补齐nginx代理配置、systemd服务、备份脚本和更新脚本。
  - 实现`--check-config`和本机管理员创建命令。
  - 部署前强制备份SQLite数据库和上传目录。
  - 在临时目录完成恢复演练，验证板块、帖子详情和图片地址可用。
